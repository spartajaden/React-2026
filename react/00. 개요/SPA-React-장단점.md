# SPA (Single Page Application)란?

## SPA 정의

**SPA (Single Page Application)** 는 **하나의 HTML 페이지**로 전체 웹 애플리케이션을 실행하는 방식입니다.

전통적인 MPA(Multi Page Application)는 페이지 이동 시마다 서버에서 새 HTML을 받아와서 화면이 깜빡입니다.

하지만 **SPA**는 처음 한 번만 HTML을 로드하고, 이후에는 JavaScript가 DOM을 직접 조작하여 화면을 갱신합니다.

> 즉, **페이지 이동 = 서버 요청**이 아니라 **페이지 이동 = JS가 화면 교체**입니다.

---

## SPA 동작 방식 (단순 비교)

| 구분 | MPA (전통적 방식) | SPA (React 방식) |
|------|-------------------|------------------|
| 페이지 이동 | 서버에 새 HTML 요청 | JS로 DOM만 교체 |
| 화면 깜빡임 | 있음 | 없음 |
| 서버 부하 | 높음 (HTML 매번 생성) | 낮음 (API만 응답) |
| 사용자 경험 | 페이지 전환 시 대기 | 부드러운 전환 |

---

## SPA의 장점

1. **빠른 화면 전환** - 서버 왕복 없이 즉시 전환
2. **부드러운 UX** - 네이티브 앱처럼 동작
3. **서버 부하 감소** - HTML 렌더링 부담이 없음
4. **프론트/백엔드 분리** - 독립적 개발 가능

## SPA의 단점

1. **초기 로딩 속도** - 첫 로딩 시 모든 JS 다운로드 필요
2. **SEO 취약** - 검색 엔진이 JS 렌더링 결과를 잘 읽지 못함 (SSR로 보완)
3. **JS 필수** - JS 비활성화 시 동작 불가
4. **메모리 관리 필요** - 장시간 사용 시 메모리 누수 위험

---

# React란?

**React**는 **Facebook(Meta)이 개발한 UI 라이브러리**입니다.

SPA를 구현하는 대표적인 도구로, **컴포넌트(Component)** 단위로 화면을 구성합니다.

> React는 프레임워크가 아니라 **라이브러리**입니다.
> (라우팅, 상태 관리 등은 별도 라이브러리 필요)

---

## React의 핵심 개념

| 개념 | 설명 |
|------|------|
| **컴포넌트** | UI를 재사용 가능한 단위로 분리 |
| **가상 DOM** | 변경된 부분만 실제 DOM에 반영 (성능 최적화) |
| **JSX** | JS에서 HTML 문법을 사용할 수 있는 확장 문법 |
| **단방향 데이터 흐름** | 부모 → 자식으로만 데이터 전달 (예측 쉬움) |
| **State** | 컴포넌트의 동적 데이터, 변경 시 자동 리렌더링 |

---

## React의 장점

### 1. 컴포넌트 재사용
- UI를 작은 단위로 나누어 여러 곳에서 재사용
- 유지보수와 테스트가 쉬움

### 2. 선언형 코딩
- "어떻게 그릴지"가 아니라 "무엇을 그릴지" 작성
- 코드가 직관적이고 가독성이 좋음

### 3. 가상 DOM 성능
- 변경된 부분만 실제 DOM에 반영 → 빠른 렌더링
- 불필요한 렌더링 방지 도구 제공 (React.memo, useMemo)

### 4. 거대한 생태계
- React Router (라우팅)
- Redux, Zustand (상태 관리)
- React Query (데이터 페칭)
- Next.js (풀스택 프레임워크)

### 5. 강력한 커뮤니티
- 전 세계 1위 UI 라이브러리
- 풍부한 자료, 튜토리얼, 오픈소스

### 6. 크로스 플랫폼
- React DOM (웹)
- React Native (모바일 앱)
- Electron (데스크톱 앱)

---

## React의 단점

### 1. 학습 곡선
- JSX, 상태 관리, 라우팅, 훅(Hook) 등 학습 필요
- 초보자에게 진입 장벽이 있음

### 2. 프레임워크가 아님
- 라우팅, 상태 관리, HTTP 통신 등을 직접 선택해야 함
- 선택 장애와 조합에 따른 혼란 가능

### 3. 잦은 업데이트
- v16 → v17 → v18 → v19 로 빠르게 진화
- 지속적인 학습 필요

### 4. 보일러플레이트
- 초기 설정에 필요한 코드 양이 많을 수 있음

### 5. 과도한 리렌더링 위험
- 불필요한 리렌더링이 발생할 수 있음
- 최적화 도구 사용 필요 (React.memo, useCallback, useMemo)

## 결론

- **SPA**는 부드러운 사용자 경험을 제공하는 현대적인 웹 아키텍처
- **React**는 SPA를 구현하는 가장 인기 있는 라이브러리
- 작은 프로젝트는 CDN 방식으로 간단히, 큰 프로젝트는 Vite 또는 Next.js로 시작
